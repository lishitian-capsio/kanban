title: 三层知识库 POC 架构
---
自搭的一套三层知识库 POC 记录。核心思路:分层沉淀不同形态的知识,再用意图识别智能体自动路由检索路径。

## 整体架构(三层)

1. **基础 RAG 向量层** — 用 Milvus 做文本切片,完整保留原文,作为底层数据基座,处理日常常规问答。
2. **GraphRAG 知识图谱层** — 基于本体规则抽取实体 + 关系,与原始段落双向绑定,支持复杂多跳推理。
3. **Wiki 结构化知识层** — 专门沉淀标准概念、词条释义,解决定义类问答不准的问题(含文档总结)。

三层知识再按业务目录分区管理。

## 检索编排

搭配 **DeepAgent 意图识别智能体**,自动判断检索路径:
- 用户问题先做拆解(可拆成多个子问题)
- 普通问题 → 向量检索
- 复杂推理 → 知识图谱
- 概念词条 → Wiki 知识库

全程自带**召回反思迭代**:封装多个 tool 给智能体,不断校验、补全信息,直到输出准确答案。

## 实测

问答准确率实测约 99.99%(自测口径)。

## 备注

这套三层分工(向量原文 / 图谱推理 / Wiki 定义)与 Kanban 自身的知识库(vault)理念相通 —— vault 也是按类型分层沉淀结构化知识。可作为后续知识检索方向的参考。
